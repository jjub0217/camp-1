# my-content-digest

## Step 1: Pre-Quiz (콘텐츠 보기 전 퀴즈)

### 핵심 원칙

**요약을 먼저 보여주지 않는다.** 퀴즈를 먼저 출제하여 틀린 부분에서 호기심을 만들고, 그 호기심이 이후 학습의 기억력을 강화한다 (Pretesting Effect, 9-12% 향상).

### 1-1: 핵심 주제 추출

콘텐츠를 분석하여 핵심 주제 3개를 추출한다. 추출 기준:

| 추출 대상 | 예시 |
|-----------|------|
| 저자/화자의 핵심 주장 | "AI 시대에 TDD가 더 중요해졌다" |
| 콘텐츠의 주요 원칙 | "Quiz-First 학습법의 원리" |
| 핵심 개념 또는 프레임워크 | "지니 비유의 의미" |

추출하지 **않는** 것:

| 제외 대상 | 이유 |
|-----------|------|
| 날짜, 통계 수치 | 암기 문제이지 이해 문제가 아님 |
| 지엽적 세부사항 | 핵심 메시지와 무관 |
| 단순 사실 확인 | 사고를 자극하지 않음 |

### 1-2: AskUserQuestion으로 3문제 출제

추출한 핵심 주제 3개에 대해 각각 1문제씩, 총 3문제를 AskUserQuestion으로 출제한다. 각 문제는 4지선다(label + description)로 구성한다.

```
AskUserQuestion({
  questions: [
    {
      question: "(핵심 주제 1에 대한 질문)?",
      header: "Q1",
      options: [
        { label: "(선택지 1)", description: "(부연 설명)" },
        { label: "(선택지 2)", description: "(부연 설명)" },
        { label: "(선택지 3)", description: "(부연 설명)" },
        { label: "(선택지 4)", description: "(부연 설명)" }
      ],
      multiSelect: false
    },
    {
      question: "(핵심 주제 2에 대한 질문)?",
      header: "Q2",
      options: [ ... ],
      multiSelect: false
    },
    {
      question: "(핵심 주제 3에 대한 질문)?",
      header: "Q3",
      options: [ ... ],
      multiSelect: false
    }
  ]
})
```

### 1-3: 즉시 피드백

사용자가 답변하면 각 문제에 대해 즉시 피드백을 제공한다.

정답인 경우:
```
✅ Q1 정답! (간단한 보충 설명)
```

오답인 경우:
```
❌ Q1 오답 — 정답은 "(정답)"
💡 (왜 이것이 정답인지 2-3문장 해설)
📌 이 부분을 콘텐츠에서 확인해보세요.
```

### 1-4: Pre-Quiz 결과 요약

3문제 채점 후 결과를 요약한다.

```
📊 Pre-Quiz 결과: X/3 정답
- Q1: ✅ / ❌
- Q2: ✅ / ❌
- Q3: ✅ / ❌
```

## Step 2: 선택적 콘텐츠 제공

Pre-Quiz 결과를 바탕으로, 사용자가 원하는 깊이의 콘텐츠를 선택할 수 있게 한다. 틀린 부분에 호기심이 생긴 상태에서 콘텐츠를 읽으면 기억에 더 잘 남는다.

### 2-1: AskUserQuestion으로 선택받기

```
AskUserQuestion({
  questions: [{
    question: "어떤 콘텐츠를 먼저 확인하시겠어요?",
    header: "학습 방식",
    options: [
      { label: "틀린 문제 관련 섹션만", description: "오답 부분의 원문을 찾아 해당 섹션만 제공" },
      { label: "핵심 인사이트 3개", description: "콘텐츠에서 가장 중요한 포인트 3개만 요약" },
      { label: "전체 요약 + 인사이트", description: "요약(3-5문장) + 인사이트(3개)를 종합 제공" },
      { label: "바로 본 퀴즈로", description: "콘텐츠 확인 없이 9문제 본 퀴즈에 도전" }
    ],
    multiSelect: false
  }]
})
```

### 2-2: 선택별 제공 내용

#### "틀린 문제 관련 섹션만" 선택 시

오답인 문제와 관련된 콘텐츠 원문 섹션을 찾아 제공한다.

```
🔍 오답 관련 콘텐츠

Q(번호) 관련: "(질문 요약)"
━━━━━━━━━━━━━━━━━━━━━━━━━━
(해당 섹션의 원문 또는 번역 발췌)
━━━━━━━━━━━━━━━━━━━━━━━━━━
💡 핵심: (이 섹션의 요점 1-2문장)
```

#### "핵심 인사이트 3개" 선택 시

콘텐츠에서 가장 중요한 포인트 3개를 제공한다.

```
💡 핵심 인사이트
1. (인사이트 1): (설명 2-3문장)
2. (인사이트 2): (설명 2-3문장)
3. (인사이트 3): (설명 2-3문장)
```

#### "전체 요약 + 인사이트" 선택 시

fetch-tweet/fetch-youtube에서 생성한 요약과 인사이트를 그대로 제공한다.

```
📋 요약
(3-5문장 요약)

💡 인사이트
1. 핵심 메시지: (내용)
2. 시사점: (내용)
3. 적용점: (내용)
```

#### "바로 본 퀴즈로" 선택 시

콘텐츠 제공 없이 바로 Step 3(본 퀴즈)로 넘어간다.

## Step 3: 본 퀴즈 (9문제)

3단계 난이도 × 3문제 = 총 9문제. 각 단계를 AskUserQuestion으로 출제하고, 단계 완료 후 즉시 정답/해설을 제공한다.

### 출제 원칙

- 핵심 주제만 출제한다 (지엽적 세부사항, 날짜, 통계 금지)
- Pre-Quiz와 동일한 문제를 내지 않는다
- 각 난이도에 맞는 사고 수준을 요구한다

### 3-1: 기본 (3문제) — 핵심 메시지, 주요 개념

콘텐츠의 핵심 메시지와 주요 개념을 이해했는지 확인한다.

| 문제 유형 | 질문 패턴 |
|-----------|-----------|
| 핵심 메시지 | "저자가 가장 강조한 것은?" |
| 주요 개념 | "X의 의미로 가장 적절한 것은?" |
| 핵심 원칙 | "이 콘텐츠의 핵심 원칙은?" |

```
AskUserQuestion({
  questions: [
    { question: "(기본 Q1)?", header: "기본 1", options: [...], multiSelect: false },
    { question: "(기본 Q2)?", header: "기본 2", options: [...], multiSelect: false },
    { question: "(기본 Q3)?", header: "기본 3", options: [...], multiSelect: false }
  ]
})
```

채점 후 즉시 피드백:
```
📊 기본 단계: X/3 정답
- 기본 1: ✅/❌ (해설)
- 기본 2: ✅/❌ (해설)
- 기본 3: ✅/❌ (해설)
```

### 3-2: 중급 (3문제) — 개념 간 관계, 근거 연결

개념들 사이의 관계를 파악하고, 저자의 주장과 근거를 연결할 수 있는지 확인한다.

| 문제 유형 | 질문 패턴 |
|-----------|-----------|
| 관계 파악 | "A와 B의 관계로 적절한 것은?" |
| 근거 연결 | "저자가 X를 주장한 근거는?" |
| 비교/대조 | "A와 B의 차이점으로 옳은 것은?" |

```
AskUserQuestion({
  questions: [
    { question: "(중급 Q1)?", header: "중급 1", options: [...], multiSelect: false },
    { question: "(중급 Q2)?", header: "중급 2", options: [...], multiSelect: false },
    { question: "(중급 Q3)?", header: "중급 3", options: [...], multiSelect: false }
  ]
})
```

채점 후 즉시 피드백:
```
📊 중급 단계: X/3 정답
- 중급 1: ✅/❌ (해설)
- 중급 2: ✅/❌ (해설)
- 중급 3: ✅/❌ (해설)
```

### 3-3: 심화 (3문제) — 실제 적용, 핵심 원리 확장

콘텐츠의 원리를 새로운 상황에 적용하거나 확장할 수 있는지 확인한다.

| 문제 유형 | 질문 패턴 |
|-----------|-----------|
| 적용 | "이 원칙을 X 상황에 적용하면?" |
| 확장 | "이 개념을 확장하면 어떤 결론이 나오는가?" |
| 판단 | "저자의 관점에서 X에 대해 어떻게 말할까?" |

```
AskUserQuestion({
  questions: [
    { question: "(심화 Q1)?", header: "심화 1", options: [...], multiSelect: false },
    { question: "(심화 Q2)?", header: "심화 2", options: [...], multiSelect: false },
    { question: "(심화 Q3)?", header: "심화 3", options: [...], multiSelect: false }
  ]
})
```

채점 후 즉시 피드백:
```
📊 심화 단계: X/3 정답
- 심화 1: ✅/❌ (해설)
- 심화 2: ✅/❌ (해설)
- 심화 3: ✅/❌ (해설)
```

### 3-4: 최종 결과

9문제 전체 채점 결과를 종합한다.

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📊 본 퀴즈 최종 결과: X/9 정답

기본: X/3 | 중급: X/3 | 심화: X/3

(결과에 따른 한 줄 코멘트)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

결과별 코멘트:
- 8-9/9: "훌륭합니다! 콘텐츠를 깊이 있게 이해하셨습니다."
- 5-7/9: "핵심은 잘 잡았습니다. 틀린 부분만 복습하면 완벽합니다."
- 0-4/9: "콘텐츠를 다시 한번 읽어보시면 훨씬 많은 것을 얻을 수 있습니다."

### 3-5: 후속 선택

최종 결과 후 AskUserQuestion으로 다음 행동을 선택받는다.

```
AskUserQuestion({
  questions: [{
    question: "다음으로 무엇을 하시겠어요?",
    header: "후속 선택",
    options: [
      { label: "다른 문제로 재퀴즈", description: "같은 콘텐츠에서 새로운 9문제를 출제" },
      { label: "관련 콘텐츠 추천", description: "이 주제와 관련된 트윗, 영상, 아티클을 검색" },
      { label: "종료", description: "학습을 마치고 세션 종료" }
    ],
    multiSelect: false
  }]
})
```

#### "다른 문제로 재퀴즈" 선택 시
이전에 출제하지 않은 새로운 관점에서 9문제를 다시 출제한다. Step 3을 처음부터 반복한다.

#### "관련 콘텐츠 추천" 선택 시
WebSearch로 콘텐츠의 핵심 주제와 관련된 추가 자료를 검색하여 3개를 추천한다.

```
🔗 관련 콘텐츠 추천
1. (제목) — (출처) — (한 줄 설명)
2. (제목) — (출처) — (한 줄 설명)
3. (제목) — (출처) — (한 줄 설명)
```

#### "종료" 선택 시
학습 결과를 간단히 정리하고 종료한다.

```
📝 학습 완료!
- Pre-Quiz: X/3 → 본 퀴즈: X/9
- 강점: (잘한 영역)
- 복습 추천: (틀린 문제 관련 주제)
```
