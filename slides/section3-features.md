## 1. CLAUDE.md
### 매번 말 안 해도 기억하는 팀 규칙

<br>

> **실제로 이런 거:**
> 팀에 새로 온 사람에게 "우리 팀은 이렇게 해"라고 적어준 **업무 매뉴얼**

<br>

- 프로젝트 폴더에 `CLAUDE.md` 파일 하나만 두면 끝
- Claude가 **매 대화 시작마다** 자동으로 읽음
- "보고서는 항상 PDF로", "존댓말 사용" 같은 규칙 저장

<br>

```
# CLAUDE.md 예시
- 보고서는 항상 한국어로 작성
- 표는 마크다운 테이블 사용
- 커밋 메시지는 한글로
```

<!-- 강사 노트: "CLAUDE.md는 Claude에게 주는 업무 매뉴얼입니다. 한 번 써두면 매번 같은 말 반복 안 해도 됩니다. 새 대화를 열어도 규칙을 기억하고 있어요." -->

---

### 직접 해보기

```bash
# 1단계: 내 메모리 확인하기
/memory
# → 현재 저장된 CLAUDE.md와 메모리 파일 목록 표시
# → 선택하면 에디터에서 바로 편집 가능

# 2단계: CLAUDE.md 자동 생성하기
/init
# → 프로젝트 구조를 분석해서 CLAUDE.md 자동 작성
# → 생성된 파일을 내 상황에 맞게 수정하면 끝

# 3단계: 잘 적용됐는지 확인하기
# → 새 대화를 시작하고 "내 규칙이 뭐야?" 라고 물어보기
```

<!-- 강사 노트: "/memory는 지금 Claude가 기억하고 있는 것을 보여줍니다. /init은 CLAUDE.md를 자동으로 만들어줍니다. 이 두 명령어만 알면 CLAUDE.md는 끝입니다." -->

## 2. Skill
### 자주 하는 업무를 레시피로 저장

<br>

> **실제로 이런 거:**
> 요리할 때 매번 레시피 검색 안 하고, **자주 쓰는 레시피를 냉장고에 붙여두는 것**

<br>

- 마크다운 파일에 워크플로우를 적어두면 됨
- `/이름` 으로 바로 호출 (예: `/review`, `/deploy`)
- Claude가 상황에 맞게 **알아서** 가져다 쓰기도 함

<br>

| 구분 | 예시 |
|------|------|
| 참조용 Skill | API 스타일 가이드, 코드 규칙 문서 |
| 작업용 Skill | `/sync` 데이터 동기화, `/deploy` 배포 |

<!-- 강사 노트: "Skill은 Claude에게 가르쳐둔 레시피입니다. CLAUDE.md가 '항상 기억해'라면, Skill은 '필요할 때 꺼내 써'입니다. 이번 캠프에서 /sync, /clarify, /wrap 세 가지 핵심 스킬을 직접 만들어봅니다." -->

---

### 직접 해보기

```bash
# 1단계: 사용 가능한 스킬 목록 보기
/
# → 슬래시(/) 입력하면 현재 사용 가능한 스킬 목록 표시

# 2단계: 스킬 호출해보기
/sync
# → 슬래시 + 이름으로 바로 실행
# → 이번 캠프에서 /sync, /clarify, /wrap 을 직접 만듭니다

# 3단계: 내 스킬 만들기 (마크다운 파일 하나면 끝)
# .claude/skills/my-skill/SKILL.md 파일을 만들면 됩니다
# → Claude에게 "나만의 스킬을 만들어줘" 라고 요청해도 OK
```

<!-- 강사 노트: "슬래시(/)만 치면 스킬 목록이 나옵니다. 스킬은 마크다운 파일 하나입니다. 코딩이 아니라 글쓰기예요. 여러분도 바로 만들 수 있습니다." -->

## 3. MCP
### 외부 도구에 꽂는 연결 플러그

<br>

> **실제로 이런 거:**
> 스마트폰에 **충전 케이블** 꽂듯이,
> Claude에게 Slack, Calendar, Notion **케이블**을 꽂아주는 것

<br>

- MCP = Model Context Protocol (외부 서비스 연결 규격)
- 한 번 연결하면 Claude가 **직접** 외부 도구를 사용
- 별도 코딩 없이 설정만으로 연결 가능

<br>

**연결 가능한 도구 예시:**

| 도구 | Claude가 할 수 있는 것 |
|------|----------------------|
| Slack | 메시지 읽기/보내기 |
| Google Calendar | 일정 조회/생성 |
| Notion | 페이지 읽기/수정 |
| 데이터베이스 | 데이터 조회/분석 |

<!-- 강사 노트: "MCP는 Claude의 팔을 늘려주는 겁니다. 원래 Claude는 터미널 안에서만 일하는데, MCP를 연결하면 Slack도 보고, 캘린더도 확인하고, Notion도 수정할 수 있어요. 이번 캠프 5일차에 직접 연결해봅니다." -->

---

### 직접 해보기

```bash
# 1단계: 현재 연결된 MCP 서버 확인
/mcp
# → 연결된 서버 목록과 상태 표시

# 2단계: MCP 서버 추가하기 (터미널에서)
claude mcp add --transport http notion https://mcp.notion.com/mcp
# → Notion 연결 완료! 이제 Claude가 Notion을 직접 읽고 씀

# 3단계: 연결 확인
claude mcp list
# → 추가된 서버 목록 확인
```

<!-- 강사 노트: "/mcp 명령 하나면 현재 연결 상태를 볼 수 있습니다. 서버 추가는 claude mcp add 한 줄이면 끝이에요. 캠프 5일차에 실제로 Notion, Slack 등을 연결해봅니다." -->

---

## 4. Subagent
### 부하 직원에게 일 시키기

<br>

> **실제로 이런 거:**
> 팀장이 **팀원에게 조사 업무를 맡기고**,
> 결과 보고서만 받아보는 것

<br>

- Claude가 **또 다른 Claude**를 불러서 일을 시킴
- 부하 직원(Subagent)은 독립된 공간에서 작업
- 팀장(메인 Claude)은 **요약 결과만** 받음

<br>

**왜 좋은가?**

| 직접 하면 | Subagent 쓰면 |
|-----------|---------------|
| 파일 100개 읽으면 대화가 길어짐 | 부하가 읽고 요약만 전달 |
| 한 번에 한 가지씩 | 여러 조사를 동시에 |
| 맥락이 섞임 | 각자 독립된 공간에서 깔끔하게 |

<!-- 강사 노트: "Subagent는 '야, 이거 조사해와' 하고 시키는 겁니다. 결과만 받으니까 내 대화 창이 복잡해지지 않아요. 파일 수십 개를 분석하거나, 여러 가지를 동시에 조사할 때 특히 유용합니다." -->

---

### 직접 해보기

```bash
# 1단계: 등록된 에이전트 목록 보기
/agents
# → 내장 에이전트 + 커스텀 에이전트 목록 표시

# 2단계: 자연어로 Subagent 사용하기
# Claude에게 이렇게 말해보세요:
"이 프로젝트의 README 파일들을 모두 찾아서 요약해줘"
# → Claude가 자동으로 Subagent를 띄워서 조사 후 결과만 전달

# 3단계: 백그라운드 실행 (기다리지 않고 다른 일 하기)
# 실행 중에 Ctrl+B → 백그라운드로 전환
# 또는: "백그라운드에서 실행해줘" 라고 말하기
```

<!-- 강사 노트: "/agents로 어떤 에이전트가 있는지 볼 수 있습니다. 직접 명령어를 외울 필요 없어요. 그냥 '조사해줘'라고 하면 Claude가 알아서 Subagent를 씁니다. Ctrl+B로 백그라운드 전환하면 기다리지 않고 다른 작업을 할 수 있어요." -->

---

## 5. Agent Teams
### 여러 부하 직원이 동시에 일하기

<br>

> **실제로 이런 거:**
> 프로젝트에서 **디자인팀, 개발팀, QA팀**이
> 각자 일하면서 서로 소통하는 것

<br>

- Subagent의 진화형: 여러 Agent가 **서로 대화하며** 협업
- 각 Agent는 완전히 **독립된 세션**
- 공유 작업 목록(Task)으로 진행 상황 조율

<br>

| Subagent | Agent Teams |
|----------|-------------|
| 팀장에게만 보고 | 팀원끼리 직접 소통 |
| 단순 조사/분석 | 복잡한 협업 프로젝트 |
| 비용 낮음 | 비용 높음 (각각 독립 세션) |

<!-- 강사 노트: "Agent Teams는 실험적 기능인데, 방향성을 이해하면 좋습니다. Subagent가 '시키고 결과 받기'라면, Agent Teams는 '각자 역할을 맡아서 알아서 협업'입니다. 예를 들어 보안 검토, 성능 검토, 테스트를 동시에 돌릴 수 있어요." -->

---

### 직접 해보기

```bash
# 1단계: 자연어로 팀 만들기
# Claude에게 이렇게 말해보세요:
"에이전트 팀을 만들어서 이 프로젝트를 분석해줘.
한 명은 코드 품질, 한 명은 보안, 한 명은 성능을 봐줘"
# → Claude가 3명의 팀원을 만들고 각자 분석 후 종합 보고

# 2단계: 팀 현황 확인
# "현재 진행 상황 알려줘" 라고 물어보기
# 또는: Shift+Up/Down으로 팀원 간 전환

# 3단계: 팀원에게 직접 지시
"보안 담당 팀원에게: 인증 부분을 집중적으로 봐줘"
# → 특정 팀원에게 추가 지시 가능
```

<!-- 강사 노트: "Agent Teams는 코딩 없이 자연어로 팀을 만듭니다. '3명으로 팀 꾸려서 분석해줘'라고 하면 됩니다. 비용이 높으니 꼭 필요할 때만 쓰세요. 실험적 기능이라 캠프에서는 맛보기만 합니다." -->

## 6. Hook
### 자동으로 실행되는 체크리스트

<br>

> **실제로 이런 거:**
> 퇴근 전 **자동으로 불 끄고, 문 잠그는** 스마트홈 루틴

<br>

- 특정 이벤트가 발생하면 **자동으로** 스크립트 실행
- Claude의 판단이 아닌 **100% 확실한 규칙** 실행
- Claude가 개입하지 않아도 알아서 동작

<br>

**Hook이 실행되는 타이밍:**

| 이벤트 | Hook 동작 예시 |
|--------|---------------|
| 파일 수정 후 | 자동으로 맞춤법 검사 |
| 대화 종료 시 | 작업 요약을 음성으로 읽어주기 |
| 코드 저장 시 | 자동 포맷팅 실행 |

<!-- 강사 노트: "Hook은 Claude가 '생각해서 하는 것'이 아니라 '무조건 실행되는 규칙'입니다. 예를 들어 대화가 끝날 때마다 요약을 읽어주거나, 파일을 수정할 때마다 자동으로 포맷을 맞추는 식이죠." -->

---

### 직접 해보기

```bash
# 1단계: 현재 설정된 Hook 확인
/hooks
# → 등록된 Hook 목록 표시 + 추가/삭제 메뉴

# 2단계: Hook 추가하기 (대화형 메뉴에서)
/hooks
# → "Add hook" 선택
# → 이벤트 선택 (예: Stop = 대화 끝날 때)
# → 실행할 명령어 입력

# 3단계: 예시 - 대화 끝나면 Mac이 말로 알려주기
# 이벤트: Stop
# 명령어: say "작업이 완료되었습니다"
# → 이제 Claude가 응답을 마칠 때마다 음성 알림!
```

<!-- 강사 노트: "/hooks 명령 하나면 Hook을 보고, 추가하고, 삭제할 수 있습니다. 코드를 직접 쓸 필요 없이 메뉴에서 선택하면 됩니다. Mac의 say 명령으로 음성 알림을 만드는 건 가장 쉬운 Hook 예시입니다." -->

---

## 7. Plugin
### 기능들을 패키지로 묶어 공유

<br>

> **실제로 이런 거:**
> 스마트폰 **앱스토어**에서 앱 하나 설치하면
> 여러 기능이 한 번에 들어오는 것

<br>

- Skill + Hook + MCP + Subagent를 **하나로 묶은 패키지**
- 설치 한 번이면 관련 기능이 통째로 들어옴
- 다른 사람이 만든 Plugin을 **Marketplace**에서 설치 가능

<br>

```
Plugin 하나에 들어있는 것들:
├── Skill     ← 워크플로우 레시피
├── Hook      ← 자동화 규칙
├── MCP       ← 외부 연결
└── Subagent  ← 전용 부하 직원
```

<!-- 강사 노트: "Plugin은 지금까지 본 기능들을 선물 세트처럼 묶어놓은 겁니다. 누군가 잘 만들어둔 Plugin을 설치하면 Skill, Hook, MCP가 한 번에 세팅됩니다. 앱스토어에서 앱 하나 깔면 여러 기능이 한 번에 들어오는 것처럼요." -->

---

### 직접 해보기

```bash
# 1단계: Plugin 메뉴 열기
/plugin
# → Discover(발견) / Installed(설치됨) / Marketplaces 탭 표시

# 2단계: Plugin 설치하기 (Discover 탭에서 선택)
# → 원하는 Plugin을 찾아서 Install 클릭
# → 또는: /plugin install <plugin-name>
# → Skill + Hook + MCP가 자동으로 세팅

# 3단계: 설치 확인
/plugin
# → Installed 탭에서 설치된 Plugin 확인
```

<!-- 강사 노트: "/plugin 명령으로 앱스토어처럼 Plugin을 찾고 설치할 수 있습니다. /plugin install 한 줄이면 Skill, Hook, MCP가 한 번에 깔립니다. 지금까지 배운 기능들이 Plugin 하나에 다 들어있는 겁니다." -->

## 컨텍스트 로딩
### 각 기능이 Claude에게 전달되는 방식

<br>

![컨텍스트 로딩 다이어그램](images/context-loading.svg)

<br>

| 기능 | 언제 로드? | 무엇이 로드? |
|------|-----------|-------------|
| **CLAUDE.md** | 세션 시작 | 전체 내용, 항상 유지 |
| **Skill** | 호출 시 | 평소엔 이름만, 호출하면 전체 |
| **MCP** | 세션 시작 | 도구 목록과 스키마 |
| **Subagent** | 생성 시 | 독립된 새 공간 |
| **Hook** | 이벤트 발생 시 | 외부에서 실행 (비용 0) |

## 기능 결합 패턴
### 실전에서는 섞어서 쓴다

<br>

하나만 쓰는 게 아니라, **조합해서** 시너지를 만듭니다

<br>

| 조합 | 이렇게 동작 | 실전 예시 |
|------|-----------|----------|
| **Skill + MCP** | Skill이 사용법 안내, MCP가 실행 | DB 스키마 Skill + DB 연결 MCP |
| **Skill + Subagent** | Skill이 작업 지시, Subagent가 수행 | `/review` Skill이 3개 Subagent로 리뷰 |
| **CLAUDE.md + Skill** | CLAUDE.md에 규칙, Skill에 상세 참조 | "API 규칙 따라" + API 가이드 Skill |
| **Hook + MCP** | Hook이 이벤트 감지, MCP로 외부 알림 | 파일 수정 시 Slack 알림 자동 발송 |

<br>

> 이번 캠프에서 여러분은
> **CLAUDE.md + Skill + MCP** 조합을 직접 구축합니다
