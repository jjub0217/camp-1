# 에이전트 루프 = 일 잘하는 신입의 업무 방식

### 1단계: 컨텍스트 수집
> "먼저 관련 자료를 찾아봅니다"

### 2단계: 작업 수행
> "이해한 내용을 바탕으로 실행합니다"

### 3단계: 결과 검증
> "제가 한 일이 맞는지 확인합니다"

**완료될 때까지 이 세 단계를 반복합니다**

# 두 가지 엔진

## 모델: Sonnet vs Opus

| | **Sonnet** | **Opus** |
|---|---|---|
| 속도 | 빠름 | 상대적으로 느림 |
| 강점 | 일상적인 작업 | 복잡한 판단이 필요한 작업 |
| 비유 | 민첩한 실무자 | 깊이 생각하는 시니어 |

**세션 중 `/model`로 언제든 전환 가능**

# 도구: Claude를 에이전트로 만드는 것

> 도구가 없으면 Claude는 **말만 할 수 있습니다**
> 도구가 있으면 Claude는 **직접 행동합니다**

<!-- 도구의 존재가 "챗봇"과 "에이전트"의 결정적 차이입니다. 일반 ChatGPT는 텍스트만 출력하지만, Claude Code는 도구를 통해 실제로 파일을 만들고 명령을 실행합니다. -->

---

# 도구 4가지 범주

| 범주 | Claude가 할 수 있는 것 |
|------|----------------------|
| **파일 작업** | 파일 읽기, 편집, 새로 만들기 |
| **검색** | 파일 찾기, 내용 검색, 코드 탐색 |
| **실행** | 명령 실행, 서버 시작, 테스트, git |
| **웹** | 웹 검색, 문서 가져오기, 오류 조회 |

### 예시: "실패한 테스트를 수정해"

1. 테스트 **실행** → 뭐가 실패했는지 확인
2. 오류 메시지 **읽기**
3. 관련 파일 **검색**
4. 코드 **읽고** 이해
5. 파일 **편집**하여 수정
6. 다시 **실행**하여 검증

<!-- 각 도구 범주를 하나씩 짚으며, "파일 작업 = 손", "검색 = 눈", "실행 = 발", "웹 = 인터넷 연결"로 비유해 주세요. 예시를 통해 도구들이 에이전트 루프 안에서 어떻게 조합되는지 보여줍니다. -->

# Claude가 접근할 수 있는 것

`claude`를 실행하면, Claude는 이것들을 볼 수 있습니다:

- **프로젝트 파일** — 현재 폴더와 하위 폴더 전체
- **터미널** — 명령줄에서 할 수 있는 모든 것
- **git 상태** — 브랜치, 변경 사항, 커밋 기록
- **CLAUDE.md** — 매 세션마다 읽는 프로젝트 지침서

> 열려 있는 파일 하나만 보는 도구와는 다릅니다.
> Claude Code는 **프로젝트 전체**를 봅니다.

<!-- "인라인 코드 어시스턴트(Copilot 등)는 현재 열린 파일만 봅니다. Claude Code는 프로젝트 전체를 봅니다. 그래서 '인증 버그 수정해'라고만 해도 관련 파일을 직접 찾아서 고칠 수 있습니다." -->

# 세션: 대화의 단위

### 세션은 임시적입니다

- 각 세션은 **새로 시작**합니다
- 지난주에 뭘 했는지 **기억하지 않습니다**
- 세션 사이에 기억시키고 싶은 것은 → **CLAUDE.md에 기록**

### 세션은 이어갈 수 있습니다

```
claude --continue     # 이전 세션 이어서
claude --resume       # 목록에서 골라서
```

![세션 연속성](images/session-continuity.svg)

# 체크포인트: 언제든 되돌리기

> Claude가 파일을 편집하기 **전에**
> 자동으로 스냅샷을 찍어둡니다

### 되돌리는 방법
- **Esc 두 번** — 바로 이전 상태로 복원
- Claude에게 **"취소해"** — 말로도 됩니다

### 되돌릴 수 없는 것
- 외부 시스템에 영향을 주는 작업 (API 호출, 배포 등)
- 그래서 Claude는 **위험한 명령 전에 반드시 물어봅니다**

<!-- "Git의 undo와 비슷하지만, Git과 별개입니다. 파일 편집에 대한 안전망이라고 생각하세요." 체크포인트가 로컬이라는 점, 외부 작업은 되돌릴 수 없다는 점을 강조하세요. -->

# 원칙 2: 구체적으로 시작하세요

### 모호하게 (작동하지만 느림)
```
> 버그 수정해
```

### 구체적으로 (빠르고 정확)
```
> 체크아웃 흐름이 만료된 카드를 가진 사용자에게
> 손상되었습니다. src/payments/를 확인하세요.
> 먼저 실패하는 테스트를 작성한 후 수정하세요.
```

> 파일 위치, 제약 조건, 원하는 방식을 알려주세요

<!-- "구체적인 프롬프트 = 더 적은 반복. 어디를 봐야 하는지, 어떻게 검증할지를 같이 알려주면 Claude가 훨씬 빨리 정확한 결과를 냅니다." -->

# 원칙 3: 지시하지 말고, 위임하세요

### 지시 (마이크로매니징)
```
> src/auth.js의 42번 줄을 열어서
> validateToken 함수를 찾아서
> 리턴값을 true에서 false로 바꿔
```

### 위임 (신뢰)
```
> 만료된 토큰으로 로그인이 되는 버그가 있어.

```

> **목표와 맥락을 주고, 방법은 Claude에게 맡기세요**

# 정리: Claude Code 작동 방식

| 개념 | 핵심 |
|------|------|
| 에이전트 루프 | 수집 → 실행 → 검증, 반복 |
| 모델 | Sonnet(빠른 실무) / Opus(깊은 사고) |
| 도구 | 파일, 검색, 실행, 웹 — 행동하게 만드는 것 |
| 접근 범위 | 프로젝트 전체 + 터미널 + git |
| 세션 | 임시적, 재개/포크 가능 |
| 안전장치 | 체크포인트 + 권한 모드 |
| 사용법 | 대화하듯, 구체적으로, 위임하듯 |
