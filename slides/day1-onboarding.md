# 왜 터미널인가?

터미널은 **컴퓨터에게 직접 말을 거는 창**입니다.
Claude Code가 여러분 대신 일하려면
이 창이 필요합니다.

> 버튼을 하나하나 클릭하는 대신
> 말로 시키면 Claude가 **직접 실행**합니다.
> 그 실행이 일어나는 곳이 터미널입니다.

<!--
강사 노트:
- "터미널은 무서운 게 아니라, 컴퓨터와 대화하는 채팅창입니다"
- "여러분이 카톡으로 부탁하듯이, Claude에게 터미널로 부탁하는 겁니다"
- Claude Code가 여러분 대신 파일 정리, 보고서 작성, 메시지 발송 등을 하려면 터미널이 필요
- 이것이 Claude Code가 터미널 기반인 이유
-->

# Claude Code란?

### 여러분의 업무를 대신 실행하는 AI 동료

> 말로 시키면, Claude가 직접 실행합니다.

<!--

# Claude Code가 해주는 4가지

|  | 할 수 있는 것 |
|---|---|
| **만들기** | "주간 보고서 만들어줘" → 데이터 수집 + 정리 + 문서 생성 |
| **고치기** | "이 리포트 양식 틀렸어" → 원인 파악 + 수정 |
| **찾기** | "지난 달 마케팅 성과 요약해줘" → 데이터 탐색 + 정리 |
| **자동화** | "매주 월요일 이 보고서 만들어줘" → 반복 업무 자동화 |

# 왜 Claude Code인가?

### 2. 직접 조치를 취한다

다른 AI는 **답변**만 합니다.
Claude Code는 **파일을 정리하고, 메시지를 보내고, 문서를 만듭니다**.

> 말만 하는 컨설턴트 vs 직접 일하는 실행자

<!--

# 왜 Claude Code인가?

### 3. 조합의 힘

여러 도구를 **연결**해서 나만의 워크플로우를 만든다.

> Slack에서 메시지 수집 + 캘린더 일정 확인 + 보고서 작성
> → 한 번에 정리

레고 블록처럼, 원하는 대로 조합.

# 에이전트 루프 = 일 잘하는 신입의 업무 방식

### 1단계: 조사
> "먼저 관련 자료를 찾아봅니다"

### 2단계: 실행
> "이해한 내용을 바탕으로 직접 합니다"

### 3단계: 확인
> "제가 한 일이 맞는지 검증합니다"

**완료될 때까지 이 세 단계를 반복합니다**

<!-- 핵심: Claude는 한 번에 끝내는 게 아닙니다. "조사 → 실행 → 확인"을 반복합니다. 마치 일 잘하는 신입이 "이거 맞나요?" 하고 스스로 검증하는 것과 같습니다. 언제든 Esc를 눌러 중단하고 방향을 바꿀 수 있다는 점도 강조하세요. -->

# 두 가지 모드

## 빠른 모드 vs 깊은 사고 모드

| | **빠른 모드 (Sonnet)** | **깊은 사고 모드 (Opus)** |
|---|---|---|
| 속도 | 빠름 | 상대적으로 느림 |
| 적합한 업무 | 일상적인 업무 | 중요한 판단이 필요한 업무 |
| 비유 | 민첩한 실무자 | 깊이 생각하는 시니어 |

**세션 중 `/model`로 언제든 전환 가능**

# 도구 4가지 범주

| 범주 | Claude가 할 수 있는 것 | 비유 |
|------|----------------------|------|
| **문서 읽고 쓰기** | 파일 읽기, 편집, 새로 만들기 | 손 |
| **자료 찾기** | 파일 찾기, 내용 검색, 데이터 탐색 | 눈 |
| **명령 실행하기** | 명령 실행, 도구 연동, 작업 처리 | 발 |
| **인터넷 검색** | 웹 검색, 최신 정보 가져오기 | 인터넷 |

# 예시: "이번 주 캠페인 보고서 만들어줘"

1. 관련 데이터 **찾기** → GA, 광고 대시보드 확인
2. 수치 **읽기** → 클릭수, 전환율, 비용 파악
3. 기존 템플릿 **검색** → 이전 보고서 양식 확인
4. 보고서 **작성** → 문서 생성
5. 팀에 **공유** → 슬랙 발송

> 도구들이 에이전트 루프 안에서 자동으로 조합됩니다

# Claude가 접근할 수 있는 것

`claude`를 실행하면, Claude는 이것들을 볼 수 있습니다:

- **프로젝트 파일** — 현재 폴더와 하위 폴더 전체
- **터미널** — 명령을 실행할 수 있는 모든 것
- **변경 기록** — 누가 언제 뭘 바꿨는지
- **CLAUDE.md** — 매 세션마다 읽는 업무 지침서

> 파일 하나만 보는 도구와는 다릅니다.
> Claude Code는 **프로젝트 전체**를 봅니다.

<!-- "일반 AI는 여러분이 복사해서 붙여넣은 내용만 봅니다. Claude Code는 폴더 전체를 봅니다. 그래서 '지난 달 보고서 찾아서 이번 달 양식으로 바꿔줘'라고만 해도 관련 파일을 직접 찾아서 처리할 수 있습니다." -->

# 세션: 대화의 단위

### 세션은 임시적입니다

- 각 세션은 **새로 시작**합니다
- 지난주에 뭘 했는지 **기억하지 않습니다**
- 기억시키고 싶은 것은 → **CLAUDE.md에 기록**

> CLAUDE.md = Claude의 **영구 기억**

<!-- "Claude Code는 세션 간 기억이 없습니다. 하지만 CLAUDE.md에 적어두면 매번 읽습니다. 이것이 '영구 기억'입니다." -->

# 체크포인트: 언제든 되돌리기

### 되돌리는 방법

# 원칙 2: 구체적으로 시작하세요

### 모호하게 (작동하지만 느림)
```
> 보고서 써줘
```

### 구체적으로 (빠르고 정확)
```
> 지난 달 마케팅 캠페인 성과를 GA 데이터 기반으로
> 3페이지 보고서로 만들어줘.
> 채널별 비교 표 포함하고, 핵심 수치는 굵게 표시해.
```

> 데이터 출처, 분량, 원하는 형식을 알려주세요

# Part 3
## Claude Code의 7가지 무기

<!-- Section 3: 주요 기능 -->

## 1. CLAUDE.md
### 매번 말 안 해도 기억하는 팀 규칙

> 팀에 새로 온 사람에게 적어주는 **업무 매뉴얼**

- 프로젝트 폴더에 `CLAUDE.md` 파일 하나만 두면 끝
- Claude가 **매 대화 시작마다** 자동으로 읽음

```
# CLAUDE.md 예시
- 보고서는 항상 한국어로 작성
- 표는 마크다운 테이블 사용
```

<!-- 강사 노트: "CLAUDE.md는 Claude에게 주는 업무 매뉴얼입니다. 한 번 써두면 매번 같은 말 반복 안 해도 됩니다." -->

---

## 2. Skill
### 자주 하는 업무를 레시피로 저장

> **냉장고에 붙여둔 자주 쓰는 레시피**

- `/이름` 으로 바로 호출 (예: `/sync`, `/deploy`)
- Claude가 상황에 맞게 **알아서** 가져다 쓰기도 함

| 구분 | 예시 |
|------|------|
| 참조용 | API 스타일 가이드, 코드 규칙 문서 |
| 작업용 | `/sync` 데이터 동기화, `/deploy` 배포 |

> 이번 캠프에서 `/sync`, `/clarify`, `/wrap` 3개를 직접 만듭니다

<!-- 강사 노트: "Skill은 Claude에게 가르쳐둔 레시피입니다. CLAUDE.md가 '항상 기억해'라면, Skill은 '필요할 때 꺼내 써'입니다." -->

---

## 3. MCP
### 외부 도구에 꽂는 연결 플러그

> Claude에게 Slack, Calendar, Notion **케이블**을 꽂아주는 것

- 한 번 연결하면 Claude가 **직접** 외부 도구를 사용
- 별도 코딩 없이 설정만으로 연결 가능

| 도구 | 할 수 있는 것 |
|------|-------------|
| Slack | 메시지 읽기/보내기 |
| Calendar | 일정 조회/생성 |
| Notion | 페이지 읽기/수정 |
| DB | 데이터 조회/분석 |

<!-- 강사 노트: "MCP는 Claude의 팔을 늘려주는 겁니다. 캠프 5일차에 직접 연결해봅니다." -->

---

## 4. Subagent
### 부하 직원에게 일 시키기

> **팀원에게 조사 맡기고, 결과 보고서만 받기**

- Claude가 **또 다른 Claude**를 불러서 일을 시킴
- 부하 직원은 독립된 공간에서 작업 → **요약만** 전달

| 직접 하면 | Subagent 쓰면 |
|-----------|---------------|
| 대화가 길어짐 | 요약만 전달 |
| 한 번에 하나씩 | 동시에 여러 개 |
| 맥락이 섞임 | 독립된 공간 |

<!-- 강사 노트: "Subagent는 '야, 이거 조사해와' 하고 시키는 겁니다. 결과만 받으니까 대화 창이 복잡해지지 않아요." -->

---

## 5. Agent Teams
### 여러 부하 직원이 동시에 일하기

> **시장 조사팀, 보고서팀, 발표자료팀**이 각자 일하면서 서로 소통

- 여러 Agent가 **서로 대화하며** 협업
- 공유 작업 목록(Task)으로 진행 상황 조율

| Subagent | Agent Teams |
|----------|-------------|
| 팀장에게만 보고 | 팀원끼리 직접 소통 |
| 단순 조사/분석 | 복잡한 협업 |
| 비용 낮음 | 비용 높음 |

<!-- 강사 노트: "Subagent가 '시키고 결과 받기'라면, Agent Teams는 '시장 조사, 보고서, 발표자료를 각각 맡아서 동시에 작업'입니다." -->

---

## 6. Hook
### 자동으로 실행되는 체크리스트

> 퇴근 전 **자동으로 불 끄고, 문 잠그는** 스마트홈 루틴

- 특정 이벤트 → **자동으로** 스크립트 실행
- Claude의 판단이 아닌 **100% 확실한 규칙**

| 이벤트 | Hook 동작 예시 |
|--------|---------------|
| 파일 수정 후 | 자동 맞춤법 검사 |
| 대화 종료 시 | 요약을 음성으로 읽기 |
| 코드 저장 시 | 자동 포맷팅 |

<!-- 강사 노트: "Hook은 '무조건 실행되는 규칙'입니다." -->

---

## 7. Plugin
### 기능들을 패키지로 묶어 공유

> **앱스토어**에서 앱 하나 깔면 여러 기능이 한 번에

- Skill + Hook + MCP + Subagent를 **하나로 묶은 패키지**
- **Marketplace**에서 다른 사람이 만든 Plugin 설치 가능

```
Plugin = Skill + Hook + MCP + Subagent
         선물 세트처럼 한 번에 설치
```

<!-- 강사 노트: "Plugin은 지금까지 본 기능들을 선물 세트처럼 묶어놓은 겁니다." -->
