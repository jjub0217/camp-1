# Claude Code란?

### 터미널에서 실행되는 AI 코딩 파트너

> 아이디어를 말하면, Claude가 코드를 짭니다.

<!--
강사 노트:
- "ChatGPT 같은 채팅창이 아닙니다. 여러분의 컴퓨터에서 직접 일하는 AI입니다."
- 핵심 비유: "비서가 여러분 옆에 앉아서 컴퓨터를 직접 조작하는 것"
- Claude Code Max 구독이 있으면 바로 시작 가능

# 설치하기

## Mac / Linux

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

## Windows (PowerShell)

```powershell
irm https://claude.ai/install.ps1 | iex
```

한 줄이면 끝. 자동 업데이트됩니다.

<!--
강사 노트:
- 터미널 열기부터 같이 시작 (Mac: Cmd+Space → "터미널", Windows: PowerShell 검색)
- 복사-붙여넣기로 충분. 외울 필요 없음
- 설치 후 터미널을 한 번 껐다 켜야 할 수도 있음 (source 안내)
- "이 한 줄이 여러분의 업무 방식을 바꿀 겁니다"
-->

# 첫 실행

```bash
cd my-project
claude
```

1. 프로젝트 폴더로 이동
2. `claude` 입력
3. 처음이면 로그인 → 끝!

<!--
강사 노트:
- cd = "change directory" (폴더 이동 명령어)
- 프로젝트 폴더가 없으면 mkdir로 하나 만들기
- 로그인은 브라우저가 자동으로 열림 → 클릭 한 번
- 실습: 참가자들과 함께 첫 실행까지 진행
-->

# Claude Code가 해주는 4가지

|  | 할 수 있는 것 |
|---|---|
| **만들기** | "로그인 페이지 만들어줘" → 계획 + 코드 + 검증 |
| **고치기** | 에러 메시지 붙여넣기 → 원인 분석 + 수정 |
| **찾기** | "이 코드 뭐 하는 거야?" → 전체 구조 파악 |
| **자동화** | 반복 작업 → 한 번에 처리 |

<!--
강사 노트:
- 각각 한 문장으로 설명
- "만들기"가 가장 임팩트 있음 → 데모로 보여주면 좋음
- 비개발자 입장: "찾기"가 가장 먼저 체감됨 (기존 코드 이해)
- MCP 연결하면 Google Drive, Slack 등 외부 데이터도 접근 가능 (Day 5 예고)
-->

# 왜 Claude Code인가?

### 1. 터미널에서 작동한다

또 다른 앱을 깔 필요 없습니다.
이미 있는 곳에서 바로 시작.

<!--
강사 노트:
- "새 앱 설치 → 로그인 → 설정 → 학습... 이런 거 없습니다"
- 터미널 = 컴퓨터와 가장 직접적으로 소통하는 방법
- 개발자들이 터미널을 쓰는 이유: 빠르고, 정확하고, 조합 가능
-->

---

# 왜 Claude Code인가?

### 2. 직접 조치를 취한다

다른 AI는 **답변**만 합니다.
Claude Code는 **파일을 편집하고, 명령을 실행하고, 커밋을 생성**합니다.

> 말만 하는 컨설턴트 vs 직접 일하는 실행자

<!--
강사 노트:
- ChatGPT: "이렇게 하세요" (복사-붙여넣기 필요)
- Claude Code: 직접 파일 수정, 테스트 실행, git 커밋까지
- 비유: "레시피를 알려주는 AI vs 직접 요리해주는 AI"
- MCP 연결 시: Jira 티켓 업데이트, Google Drive 문서 읽기도 가능
-->

# 왜 Claude Code인가?

### 3. Unix 철학 - 조합의 힘

작은 도구들을 **연결**해서 강력한 워크플로우를 만든다.

```bash
tail -f app.log | claude -p "이상 징후 발견하면 Slack으로 알려줘"
```

레고 블록처럼, 원하는 대로 조합.

<!--
강사 노트:
- Unix 철학: "한 가지를 잘 하는 작은 도구를 만들고, 파이프(|)로 연결한다"
- Claude Code도 이 철학을 따름 → 다른 도구와 자유롭게 조합 가능
- 이 개념이 Day 2-4에서 배울 Skills(/sync, /clarify, /wrap)의 기반
- 지금은 "아, 연결이 되는구나" 정도만 이해하면 OK
-->

# 어디서든 Claude Code

| 환경 | 설명 |
|------|------|
| **터미널 (CLI)** | 핵심. `claude` 한 단어로 시작 |
| **VS Code** | 에디터 안에서 바로 사용 |
| **JetBrains** | IntelliJ, PyCharm 등 |
| **웹** | claude.ai/code 에서 브라우저로 |
| **데스크톱 앱** | 독립 실행형 애플리케이션 |
| **Slack** | Slack에서 멘션으로 코딩 작업 요청 |
| **GitHub Actions** | CI/CD 자동화 |

<!--
강사 노트:
- 이번 캠프에서는 "터미널"이 메인
- VS Code/JetBrains 사용자는 IDE 확장도 설치 추천
- "어디서든"이 핵심 메시지: 하나 배우면 어디서나 쓸 수 있다
- Slack 연동은 팀 활용 시 강력 → 캠프 후반에 맛보기
-->

# Section 1 정리

1. **설치**: 터미널에서 한 줄이면 끝
2. **실행**: `cd my-project && claude`
3. **능력**: 만들기, 고치기, 찾기, 자동화
4. **철학**: 터미널 + 직접 조치 + 조합의 힘

> 지금부터 여러분의 터미널에는 AI가 상주합니다.
