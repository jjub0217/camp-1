# 삽질 자동 기록 시스템 (TroubleLog)

## 문제
> 프론트엔드 개발자가 코딩 중 겪는 삽질 과정을 자동으로 기록·분석하여, 학습 자산이자 채용 포트폴리오로 활용하고 반복되는 에러 패턴을 자동 감지하여 해결을 돕는다

- **현재 상태**: 주 5회 이상 에러/삽질을 겪지만 해결에 급급해 기록은 0건. 같은 패턴의 실수를 반복해도 알아차리지 못하고 매번 처음부터 해결
- **원하는 상태**: 삽질 발생 시 자동으로 컨텍스트가 수집되고, Notion에 트러블슈팅 문서가 생성되며, 과거 기록에서 반복 패턴이 감지되면 이전 해결법을 자동 제안
- **성공 기준**: 주 3건 이상 트러블슈팅 문서 자동 생성, 반복 에러 패턴 감지 시 이전 해결법 자동 제안으로 해결 시간 50% 단축

## 스킬
| # | 스킬명 | 한 줄 설명 | 상태 |
|---|--------|-----------|------|
| 1 | `/my-trouble-detect` | Hook으로 터미널 에러를 자동 감지하고, Context Sync 패턴으로 에러 로그·git diff·환경 정보를 멀티소스 수집 | 🔨 진행중 |
| 2 | `/my-trouble-doc` | Session Wrap 패턴으로 삽질 과정을 구조화하고, MCP(Notion)로 트러블슈팅 페이지를 자동 생성 | 🔨 진행중 |
| 3 | `/my-trouble-pattern` | History Insight + Session Analyzer 패턴으로 과거 기록을 분석하여 반복 에러 패턴을 감지하고 이전 해결법을 자동 제안 | 🔨 진행중 |
| 4 | `/my-trouble-learn` | Content Digest 패턴으로 과거 트러블슈팅 기록을 퀴즈 형태로 복습하여 같은 실수를 방지 | 🔨 진행중 |

## 변화 기록
- **Day 1**: "Claude Code로 뭘 할 수 있는지 감이 없었다"
- **Day 6**: "Hook, MCP, Context Sync, Session Wrap, History Insight, Content Digest 등 캠프에서 배운 모든 개념을 하나의 시스템으로 설계할 수 있게 됐다"
- **가장 크게 달라진 점**: 개별 스킬을 따로 배우는 것이 아니라, 전체가 연결되어 하나의 자동화 시스템을 구성할 수 있다는 것을 체감함
